## Context

Build Plan Task 3. Add a You.com live lane — the $1K prize integration. Two queries (one calm,
one trending/hot) bring real current web results into the feed, correctly shielded by tier.
Needs `VITE_YOU_API_KEY` (currently filled).

Depends on T-001-02 because both wire into `fetchLiveFeed()` in `feed.ts` — serialized to
avoid conflicting edits.

## Implementation notes

- New file `src/sources/youcom.ts`. GET `https://ydc-index.io/v1/search` with header
  `X-API-Key: ${import.meta.env.VITE_YOU_API_KEY}`, params `query` + `count=5`. Run two
  queries: calm ("wholesome calming nature") and trending ("biggest tech hot takes today").
- Map results → `FeedItem` (`source: "you.com"`). Calm query → low intensity prior; trending
  → high prior. Wrap in try/catch; return `[]` on failure.
- Add the You.com results into `fetchLiveFeed()` alongside the Reddit results; keep the same
  dedupe + ~20 cap.
- ⚠️ Key exposure: client-side is fine for the local demo. Note in the run summary that a
  public deploy needs a proxy/serverless function so the key isn't in the bundle.
- If `VITE_YOU_API_KEY` is empty: skip this lane, note it, do not fail.
