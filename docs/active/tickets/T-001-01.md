## Context

Build Plan Task 1. The readiness `score` in `App.tsx` currently comes only from the manual
slider. Pull the real daily readiness from Oura so the tier reflects the user's actual state
on load, keeping the slider as a manual override. Needs `VITE_OURA_TOKEN` (currently filled).

Reference adapter pattern: `../../oura-mcp/src/provider/oura_provider.ts`.

## Implementation notes

- New file `src/sources/oura.ts`: GET
  `https://api.ouraring.com/v2/usercollection/daily_readiness?start_date={today}&end_date={today}`
  with header `Authorization: Bearer ${import.meta.env.VITE_OURA_TOKEN}`. Take the latest
  `.data[].score`. Wrap in try/catch; return `null` on any failure.
- In `App.tsx`: on mount, if a score comes back, set `score` from Oura. The slider stays a
  manual override (user can still drag it).
- ⚠️ CORS: Oura may block browser calls. If so, add a Vite dev proxy in `vite.config.ts`
  (`server.proxy['/oura'] → https://api.ouraring.com`, rewrite to strip `/oura`) and call
  `/oura/v2/...` instead.
- If `VITE_OURA_TOKEN` is empty: skip the fetch entirely, leave the slider as the only source,
  and note it in the run summary. Do not fail the build.

## Acceptance Criteria

- [ ] `src/sources/oura.ts` exists and fetches daily readiness behind try/catch.
- [ ] With the token set, the header score reflects real Oura data on load.
- [ ] The manual slider still overrides the score after load.
- [ ] HERO_FEED + flip still work if the Oura call fails or the token is missing.
- [ ] `npx tsc -b` is clean and the dev server boots.
