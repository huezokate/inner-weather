# Wire Inner Weather's live sources, backend, and demo polish

The curated HERO_FEED + readiness slider + Sharp↔Fog flip already work with zero keys
(the demo floor). This story layers live content and persistence on top **without ever
breaking that floor**, then polishes for the hackathon demo. Each ticket = one task from
`BUILD_PLAN.md`. Every live integration must wrap fetches in try/catch and fall back when
its key is missing or the call fails.

## Track breakdown

- **Track A — readiness & persistence (App.tsx):** T-001-01 (Oura readiness) → T-001-04
  (InsForge diary). Serialized because both edit `App.tsx`.
- **Track B — live feed (feed.ts):** T-001-02 (Reddit) → T-001-03 (You.com) → T-001-05
  (AI classifier). Serialized because all three edit `fetchLiveFeed()` in `feed.ts`.
- **Polish:** T-001-06 (flip timing, beautiful Fog/Sharp states, GIF, demo script, deploy)
  runs last, after both tracks land.

## DAG

```
T-001-01 (Oura) ───────► T-001-04 (InsForge) ──┐
                                                ├─► T-001-06 (Polish + demo)
T-001-02 (Reddit) ─► T-001-03 (You.com) ─► T-001-05 (Classifier) ─┘
```

Execution waves (max_threads = 2):

## Key status (read `.env.local` at run time — this may change)

- ✅ `VITE_YOU_API_KEY`, `VITE_OURA_TOKEN` filled → T-001-01, T-001-03 run live.
- ✅ `VITE_INSFORGE_BASE_URL` / `VITE_INSFORGE_ANON_KEY` / `INSFORGE_API_KEY` filled and
  verified (backend reachable, both keys authenticate) → T-001-04 and T-001-05 run LIVE.
  The `diary` table does not exist yet — T-001-04 provisions it. Keep localStorage / source-prior
  intensities only as a runtime safety net so a failed call never breaks the demo.
